# ☕ 모카 스트리트 — 커피숍 시뮬레이터

Fast Food Simulator를 레퍼런스로 한 **1인칭 커피숍 운영 시뮬레이터**입니다.
Three.js(r158, 로컬)로 렌더링하며 빌드 없이 브라우저에서 바로 실행됩니다.

## 실행
로컬 서버로 폴더를 서빙한 뒤 `index.html`을 엽니다 (포인터 락 사용).

```
npx serve .   # 또는 python -m http.server
```

## 하루 구조 (준비 → 영업 → 정산)
- **🔧 영업 준비 (08:00, 시간 정지)** — 손님 없이 자유롭게 준비: 창고에서 **재고 보충(정상가)**, **[B] 기구 배치**, **[M] 관리 패널**에서 **장비 구매**(추가 그라인더·2번째 에스프레소 머신·추가 스티머)와 **업그레이드 구매**. `[O]`로 영업 시작
- **☕ 영업 (09:00→18:00, 빠르게)** — 손님 응대에 집중. 배치·구매는 잠금, 재고가 떨어지면 창고에서 **비상 보충(웃돈 1.8배)**만 가능
- **📊 정산** — 매출·팁·지출·**임대료**(고정 지출) 차감 후 순이익 확인. **일일 목표 순이익** 달성 시 평판 +3. 다음 날 준비로
- **⚠ 경영 위기 / 폐업** — 잔액이 마이너스면 경고, 누적 적자가 −50,000을 넘으면 폐업(게임 오버). 임대료는 날이 갈수록 오르므로 장비로 처리량을 키워 성장해야 함

## 영업 중 게임 루프
1. **주문 접수** — 줄 선 손님이 주문을 **말하면**, 계산대(ORDER)의 **POS 단말기**에 다가가 `E`로 화면을 열고(모니터 확대) 손님이 말한 **메뉴를 직접 골라 `주문 확정`**. 요청과 일치해야 접수됨(샷 추가·디저트 포함). `🔊 다시 듣기`로 주문 재생, `Esc` 취소
2. **제조** — 포터필터 분리 → 그라인더에서 원두 분쇄 → **탬핑 스테이션에서 `[E]` 꾹 눌러 게이지 채우기(퍼펙트 존에서 떼면 크레마 팁 보너스 · 완벽하지 않으면 퍽이 고르지 않아 추출 시 물총)** → 머신에 장착 → 컵 올려 샷 추출 → 넉박스에 가루 비우기 → 얼음/스팀밀크/시럽/휘핑/물로 레시피 완성 (`R` 레시피북)
   - **🩸 물총 (채널링)** — 탬핑이 **퍼펙트가 아니면** 퍽이 고르지 않아 추출 시 가는 줄기가 **비스듬히 사방으로 튄다**(스프릿징). 시각 연출로, 퍼펙트 탬핑(크레마 보너스)을 유도하는 피드백
   - **☕ 추출 컨디션 → 만족도** — 추출 상태가 손님 만족도에 직접 반영. **물총(채널링)**·**과/부족 추출(분쇄도 빗나감)**이면 팁이 깎이고, 둘 다 나쁘면 평판도 하락. 퍼펙트 탬핑 + 이상 분쇄도면 팁 보너스(+ 만족도 최상). 서빙 시 손님이 **표정(😍 만족 / 🙂 보통 / 😖 불만)과 한마디**(예: "맛있어요!" / "별로네요…")로 반응 — 인내심이 다 닳아 떠나는 손님도 😠 불평 한마디
   - **⚙️ 분쇄도 (그라인더)** — 그라인더의 **분쇄도 다이얼**을 빈손으로 `[E]`를 꾹 눌러 조정(아래=가늚, 위=굵음). 설정은 머신에 저장되며 **포터필터 소지와 무관하게** 언제든 조절 가능. 이후 빈 포터필터를 들고 `[E]`로 그 설정대로 분쇄. 분쇄도는 **추출 속도**를 정함 — **이상(초록)** 구간이면 꾸준한 줄기 + 팁 보너스, 너무 **가늘면** 추출이 막혀 **느리게 뚝뚝**(과다), 너무 **굵으면** 빨라져 **굵고 빠른 줄기**(부족). (탬핑이 완벽하면 줄기 모양은 분쇄도가, 불균일하면 물총이 우선)
   - **🍯 정량 도징 (시럽·휘핑)** — 시럽/휘핑은 컵을 들고 `[E]`를 꾹 눌러 도징 게이지를 채우고 **퍼펙트 존(정량)에서 떼면 팁 보너스**. 너무 적게 넣으면 다시 시도, 끝까지 눌러도 성공(보너스 없음). 탬핑·스티밍과 같은 게이지 메커니즘
   - **🎨 라떼아트 (자유 푸어 · 유체)** — 샷이 든 **핫 음료**에 스팀밀크를 부으면 라떼아트 미니게임이 열림. `[E]/좌클릭`을 누른 채 마우스를 **위로 밀며 좌우로 흔들면** 표면에 흐름이 생기고 흰 우유가 그 흐름에 끌려(어드벡션) 잎으로 번짐 — 마지막에 가운데로 곧게 빼면 줄기(컷스루). 흔들기·위로빼기·좌우 대칭·적정 양으로 채점. 잘 그리면 팁 보너스(+평판), 대충 부어도 음료는 완성. 아이스 음료는 그대로 즉시 붓기.
3. **서빙** — 픽업대(PICK UP)에서 `E`, 주문과 정확히 일치해야 성공
4. **보상** — 가격 + 남은 인내심에 비례한 팁, XP/평판 상승

## 조작
| 키 | 동작 |
|---|---|
| WASD / 마우스 | 이동 / 시점 |
| E · 좌클릭 | 상호작용 |
| Q | 들고 있는 것 버리기 |
| R | 레시피북 |
| B | 기구 배치 (준비 단계) |
| M / O | 관리 패널 / 영업 시작 (준비 단계) |
| Esc | 일시정지 |

## 시스템
- 메뉴 9종 음료 + 디저트 3종, 레벨업으로 순차 해금 (Lv.1~5)
- 그라인더·스티머·온수/냉수·에스프레소 머신 모두 **비동기 작업**: 컵/재료를 올려두고 다른 일을 하다가 머신 위 프로그레스 바가 "✓ 완료"가 되면 회수
- 포터필터 라이프사이클: 상태(빈 상태/원두 채움/탬핑/사용한 가루) 순환, 탬핑 스테이션에서 다지고 넉박스에서 가루 비워 재사용 (버릴 수 없는 영구 도구)
- 신규 게임 시 9단계 인터랙티브 튜토리얼 (`T`로 건너뛰기)
- 손님 인내심·평판·팁 시스템, 화난 손님은 평판 하락
- 업그레이드 6종 (듀얼 그룹헤드, 고속 추출, 자동 스티머, 인테리어, 광고, 그라인더)
- 장비 상점: 준비 단계에서 추가 그라인더·2번째 에스프레소 머신·추가 스티머를 사서 빈 카운터에 배치(병렬 작업으로 처리량 ↑), 배치·구매 내역 저장
- 경제 압박: 임대료(고정 지출, 날마다 상승) + 일일 목표 + 경영 위기·폐업, Lv6 마스터 바리스타 팁 +12%
- localStorage 자동 저장 (하루 마감 시) — `이어하기` 지원

## 파일 구조
빌드 없이 전역 IIFE 패턴으로 로드됩니다 (`index.html`의 `<script>` 순서 = 의존성 순서). 유지보수·공동작업을 위해 관심사별로 분리:
- `js/audio.js` — `AudioFX`: WebAudio 효과음 신스 (게임 로직과 독립 — 사운드만 튜닝)
- `js/data.js` — `DATA`: 레시피·디저트·업그레이드·장비·창고·경제/탬핑/라떼아트 밸런스 수치 (콘텐츠·밸런스만 튜닝)
- `js/textures.js` — 프로시저럴 캔버스 텍스처 (마루/대리석/벽돌/칠판/거리 배경)
- `js/world.js` — 카페 인테리어·머신 모델·조명·충돌 박스
- `js/customers.js` — 손님 모델·대기열 AI·인내심
- `js/player.js` — 1인칭 컨트롤러·상호작용 레이캐스트
- `js/effects.js` — `Effects`: 서빙 완료 연출(체크 팝·금색 파티클·컵 사라짐) — `scene`만 주입
- `js/ui.js` — `UI`: 표현 레이어(HUD·시계·손에든것·조준 프롬프트·레시피북·주문표 티켓) — 상태를 읽어 DOM만 그림
- `js/tutorial.js` — `Tutorial`: 신규 게임 튜토리얼(문구 + 진행 로직), 자체 상태 소유
- `js/latteart.js` — `LatteArt`: 라떼아트 자유 푸어 미니게임(작은 CPU 유체 그리드 — 피처 움직임이 속도장을 만들고 흰 우유를 어드벡션, 표면 변위로 물결 표현, 흔들기·대칭·양 채점), 자체 상태 소유
- `js/game.js` — 게임 코어: 상태·주문/제조/서빙·하루 사이클·경제·상호작용 (데이터/사운드/연출/표현/튜토리얼은 위 모듈에서 가져다 씀)
- `js/main.js` — 렌더러(ACES 톤매핑, 소프트 섀도)·증기 파티클·메인 루프

> 표현/튜토리얼 모듈은 `game.js`가 `init()`에서 주입하는 **라이브 게터 컨텍스트**(`{ S(), held(), orders(), ... }`)로 코어 상태를 읽습니다. 상태 변경·구매·저장 같은 로직은 코어에만 있습니다.
