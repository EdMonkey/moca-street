> 대상: `js/*.js` (game.js, world.js, customers.js, player.js, editor.js, main.js, textures.js)
> 범위: 정확성 결함(논리/상태/리셋/경합/저장)만. 주관적 밸런스는 별도(`balance-report.md`).

코드베이스는 전반적으로 견고하며(머신 비동기 작업·탬핑 상태머신·레시피 매칭 모두 정상), 아래 5건의 실제 결함을 발견했다. 오검 후보(XP 오프바이원, `tut` TDZ, canonical 매칭의 `perfect` 필드, counterBox 인덱싱, dualHead 잠금 게이팅)는 모두 정상으로 확인됨.

## ✅ 이번에 수정한 항목

### 1. [med] 탬핑 입력(`useDown`)이 눌린 채로 멈춤 — `game.js`
- **증상**: `[E]`/좌클릭을 누른 상태에서 포인터락이 풀리거나(Esc·[M] 패널·alt-tab) 창 포커스를 잃으면 `keyup`/`mouseup`이 발생하지 않아 `useDown`이 `true`로 남는다. 재개 시 탬핑 게이지가 입력 없이 저절로 차오르거나 자동 확정된다.
- **수정**: 입력 핸들러에 `window 'blur'`와 `pointerlockchange`(언락 시) 리스너를 추가해 `useDown=false`로 강제 해제.
- 위치: `js/game.js` 입력 설정부.

### 2. [med] 픽업 자리 부족(4 < 대기열 5) → 손님 겹침·회계 오염 — `world.js` / `customers.js`
- **증상**: `queueSpots`는 5개인데 `pickupSpots`는 4개. 픽업 자리가 모두 차면 5번째 손님의 `pickupIdx`가 `0`으로 폴백되어 기존 0번 손님과 겹치고, 이후 `pickupSpotFree()`가 0번 자리를 잘못 카운트해 회계가 꼬인다.
- **수정**: (a) `pickupSpots`에 5번째 좌표 `[-2.4, 1.6]` 추가해 대기열과 수를 맞춤. (b) 그래도 빈 자리가 없을 때의 폴백을 `0` → `-2`(유효 인덱스가 아닌 값)로 바꿔 회계 오염을 차단.

### 3. [low] 저장 일관성 — 재고 보충이 즉시 저장되지 않음 — `game.js`
- **증상**: 장비/업그레이드 구매는 `save()`를 즉시 호출하나 재고 보충(`restock`)은 안 함. 영업 중 비상 보충 후 새로고침하면 돈·재고가 되돌아가는(환불) 불일치/악용 소지.
- **수정**: `restock` 처리 끝에 `save()` 추가해 구매와 동일 정책으로 통일.

### 4. [med, cosmetic] 에스프레소 증기 이미터가 마지막 머신만 추적 — `world.js:594` / `main.js`
- `buildEspresso`가 매번 `env.machines.espressoGroup = g`로 덮어써, 2번째 에스프레소 머신을 사면 증기 파티클이 마지막에 만든 머신에서만 나온다(시각 한정).
- 권장 수정: 단일 참조 대신 이미터 그룹을 **배열**로 관리. (world.js·main.js 양쪽 수정 필요 → 별도 작업)

### 5. [med] 이어하기 시 장비 폴백 좌표 중첩 가능 — `game.js` `recreateEquipment`
- 저장 레이아웃에 매칭되는 id가 없거나 빈 공간이 소진되면 여러 장비가 동일 폴백 `{x:0, z:-4.3}`에 쌓여 상호작용 히트박스가 겹칠 수 있다(`applyLayout` 전에 `findFreeSpot`이 도므로).
- 권장 수정: 종류별로 고유한 폴백 좌표를 쓰고, id 생성을 두 경로(`buyEquipment`/`recreateEquipment)에서 결정적으로 일치시키기. 리로드 경로라 신중한 테스트 필요 → 별도 작업.
